Loudoun Decks — Website & SEO Audit
URL audited: https://ldn-decks-next.vercel.app/ Date: 2026-04-15
1. Executive summary
The site is a professional, conversion-focused marketing website for a Northern Virginia deck
construction company, built on Next.js and deployed to Vercel. Overall presentation and technical
foundation are solid, but several critical SEO bugs affect the most important pages (city-level location
pages), key schema opportunities are unused, and the blog/showcase are empty.
The three highest-priority issues:
1. Broken canonical URL on ~63 city-level location pages — they all point to /layout.js
2. Vercel staging is indexable — risk of duplicate content with the production domain
3. Missing AggregateRating and FAQPage schemas — leaving rich snippets on the table despite
having the raw data (5.0 / 41 reviews and 17 FAQs)

4. Critical bugs
4.1 Broken canonical on city-level pages
I tested 18 pages. The pattern is clear:
Page type Canonical Status
/ https://ldndecks.com OK
/services/new-decks https://ldndecks.com/services/new-decks OK
/near-you/loudoun-county (county) https://ldndecks.com/near-you/loudouncounty
OK
/near-you/loudoun-county/ashburn
(city)
https://ldndecks.com/layout.js BROKEN
/near-you/fairfax-county/vienna https://ldndecks.com/layout.js BROKEN
/near-you/prince-williamcounty/woodbridge
https://ldndecks.com/layout.js BROKEN
Root cause: the dynamic [city]/page.tsx route — generateMetadata is not receiving the segment
correctly, likely referencing params.layout or __filename incorrectly, or the function that constructs
alternates.canonical has a bug.
Affected scope: ~63 city pages (18 Loudoun + 20 Fairfax + 14 Prince William + 7 Arlington + 4 Stafford)
— the most important SEO layer for local searches.
Fix (Next.js 13+ App Router) in app/near-you/[county]/[city]/page.tsx :
export async function generateMetadata(
{ params }: { params: { county: string; city: string } }
): Promise<Metadata> {
const { county, city } = params;
return {
alternates: {
canonical: `https://ldndecks.com/near-you/${county}/${city}`,
},
openGraph: {
url: `https://ldndecks.com/near-you/${county}/${city}`,
// ...
},
};
}
4.2 Vercel staging is indexable
Confirmed:
GET https://ldn-decks-next.vercel.app/robots.txt
User-Agent: *
Allow: /
No X-Robots-Tag header is returned. Google can index the staging deployment.
Fix — add to next.config.js :
async headers() {
if (process.env.VERCEL_ENV !== 'production') {
return [{
source: '/:path*',
headers: [{ key: 'X-Robots-Tag', value: 'noindex, nofollow' }],
}];
}
return [];
}
Alternatively, a robots.ts conditioned on VERCEL_ENV .
4.3 og:url hardcoded to homepage on all pages
On /services/new-decks the meta tag is:
<meta property="og:url" content="https://ldndecks.com"/>
It should be the URL of the current page. This affects social media sharing (wrong page shared when
someone shares a service page).
4.4 og:image identical on every page
The same image ( img64.jpeg ) is used sitewide — missed opportunity for service/location-specific
preview images.
5. Schema.org — missed opportunities

6. Other issues worth fixing
Broken Facebook sameAs — LocalBusiness contains:
"sameAs": ["https://www.facebook.com/", ...]
Replace with the actual Facebook page URL, or remove it entirely.
Identical lastmod on all 107 sitemap URLs — generate lastmod from the actual content edit date,
not build time. Google discounts sitemaps with artificial timestamps.
Generic image alt text — alt="New Decks" , alt="Relaxing deck" are not optimized. Use
descriptive alts with keywords, e.g. alt="Custom Trex composite deck in Ashburn Virginia
backyard" .
No visible breadcrumbs — neither in HTML nor in schema.
No hreflang — OK since the site is English-only, but a future es-US variant could help in Northern
Virginia.
Weak internal linking between geo pages — e.g. Ashburn pages should link to Leesburg, Sterling, etc.
(neighbor cities) to distribute PageRank and help crawling.
Blog is empty — /blog is in the sitemap but has no articles. Long-tail traffic opportunities lost:
"composite vs wood deck Virginia", "cost of a deck in Ashburn", "how long does a Trex deck last", etc.
Potential "thin content" risk on geo pages — if the 63 city pages are just templates with the city name
swapped in, Google may ignore or penalize them as doorway pages. Each should contain unique content
(local projects, zoning references, area-specific details).

4. Add FAQPage schema (≈30 min) — free rich snippet
5. Per-page og:url (≈30 min)
6. Unique content on city pages — larger investment, but critical to avoid being flagged as doorway
