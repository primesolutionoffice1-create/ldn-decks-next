## 1.1 Fix broken canonical on ~63 city pages

PROBLEM: Every city-level page under /near-you/[county]/[city]/ has:
  <link rel="canonical" href="https://ldndecks.com/layout.js"/>
This is a Next.js App Router bug — the generateMetadata function is
referencing the layout filename instead of the actual route path.

AFFECTED: all pages matching /near-you/*/[city] — approximately 63 pages
(18 Loudoun + 20 Fairfax + 14 Prince William + 7 Arlington + 4 Stafford)

NOT AFFECTED: homepage, /about, /services/*, /near-you/[county] (county-level),
/showcase, /blog, /faqs, /team — these all have correct canonicals.

FIX: Create a centralized metadata builder and fix the city page metadata.

Step 1 — Create lib/seo.ts:

  import type { Metadata } from "next";

  export const SITE_URL = "https://ldndecks.com";

  export function buildMetadata({
    path,
    title,
    description,
    image = "/images/og-default.jpg",
  }: {
    path: string;
    title: string;
    description: string;
    image?: string;
  }): Metadata {
    const url = `${SITE_URL}${path}`;
    return {
      title,
      description,
      alternates: { canonical: url },
      robots: { index: true, follow: true },
      openGraph: {
        url,
        title,
        description,
        siteName: "Loudoun Decks",
        locale: "en_US",
        type: "website",
        images: [{
          url: `${SITE_URL}${image}`,
          width: 1200,
          height: 630,
          alt: title,
        }],
      },
      twitter: {
        card: "summary_large_image",
        title,
        description,
        images: [`${SITE_URL}${image}`],
      },
    };
  }

Step 2 — Fix app/near-you/[county]/[city]/page.tsx generateMetadata:

  export async function generateMetadata(
    { params }: { params: { county: string; city: string } }
  ) {
    const { county, city } = params;
    // Look up city data from your data source
    const data = getCityData(county, city);
    if (!data) return {};

    return buildMetadata({
      path: `/near-you/${county}/${city}`,
      title: `Deck Builder in ${data.name}, VA | Custom Decks | Loudoun Decks`,
      description: `Top-rated deck builder in ${data.name}, ${data.countyName}. 5.0★ Google (41 reviews). Free estimate, 5-year warranty. Call (571) 655-7207.`,
    });
  }

Step 3 — Apply the same pattern to all other dynamic routes:
  - app/services/[slug]/page.tsx → path: `/services/${slug}`
  - app/near-you/[county]/page.tsx → path: `/near-you/${county}`
  - app/blog/[slug]/page.tsx → path: `/blog/${slug}`

## 1.4 Block staging from indexing

PROBLEM: ldn-decks-next.vercel.app is publicly accessible and indexable.
Same content as ldndecks.com = duplicate content risk.

FIX Option A — noindex header for non-production (recommended):

In next.config.js (or next.config.ts):

  async headers() {
    if (process.env.VERCEL_ENV !== "production") {
      return [{
        source: "/:path*",
        headers: [
          { key: "X-Robots-Tag", value: "noindex, nofollow, noarchive" },
        ],
      }];
    }
    return [];
  },

FIX Option B — dynamic robots.ts:

Create app/robots.ts:

  import type { MetadataRoute } from "next";

  export default function robots(): MetadataRoute.Robots {
    if (process.env.VERCEL_ENV !== "production") {
      return { rules: [{ userAgent: "*", disallow: "/" }] };
    }
    return {
      rules: [{ userAgent: "*", allow: "/", disallow: "/private/" }],
      sitemap: "https://ldndecks.com/sitemap.xml",
      host: "https://ldndecks.com",
    };
  }

Apply BOTH options for belt-and-suspenders protection.

# PHASE 4 — SECURITY HEADERS (Day 3)

## 4.1 Add security headers via next.config.js

In next.config.js, inside the headers() function:

  async headers() {
    const securityHeaders = [
      { key: "X-Content-Type-Options", value: "nosniff" },
      { key: "X-Frame-Options", value: "SAMEORIGIN" },
      { key: "Referrer-Policy", value: "strict-origin-when-cross-origin" },
      { key: "Permissions-Policy", value: "geolocation=(), microphone=(), camera=()" },
      { key: "Strict-Transport-Security", value: "max-age=31536000; includeSubDomains; preload" },
      { key: "X-DNS-Prefetch-Control", value: "on" },
    ];

    const headers = [
      { source: "/:path*", headers: securityHeaders },
    ];

    // Add noindex for non-production (staging protection)
    if (process.env.VERCEL_ENV !== "production") {
      headers.push({
        source: "/:path*",
        headers: [
          { key: "X-Robots-Tag", value: "noindex, nofollow, noarchive" },
        ],
      });
    }

    return headers;
  },
