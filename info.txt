## IMPORTANT (SEO and credibility impact)

**5. Facebook link goes to generic facebook.com**
The footer has a social link pointing to `https://www.facebook.com/` — with no specific business page. A user lands on Facebook's homepage, not on Loudoun Decks' page.
→ **Fix:** Use this url https://www.facebook.com/profile.php?id=61573750423712.

**6. Social links in footer have no accessible name**
All 4 social links (Facebook, X/Twitter, Instagram, TikTok) have no text and no `aria-label`. Screen readers will read them as purposeless links.
→ **Fix:** Add `aria-label="Facebook"`, `aria-label="Instagram"`, etc. to each one.

**7. Form fields have no `<label>` or `aria-label`**
The 5 form fields (name, email, phone, select, textarea) rely solely on `placeholder`. Placeholder text disappears on focus and is not a valid substitute for accessibility.
→ **Fix:** Add visible `<label>` elements (or at minimum `aria-label`) to each field.

**8. Missing `<meta name="robots">`**
No robots meta tag is set. While search engines will index by default, it should be explicit — especially since the site is on vercel.app (staging?). If it's staging, it should be `noindex`.
→ **Fix:** Add `<meta name="robots" content="index, follow">` on production, or `noindex` on staging.

**10. H4 headings — text concatenated without space**
"Restorationand Maintenance", "Fast andReliable Services", "Skilled andTrained Experts" — they have `<br>` in the HTML, but screen readers and Google read the text as glued together. A space is missing before the `<br>`.
→ **Fix:** Change `Restoration<br>and` to `Restoration <br>and` (add a space).

## MODERATE (polish and best practices)

**12. Missing Skip Navigation link**
There's no "Skip to main content" link for keyboard accessibility. This is a WCAG 2.1 Level A requirement.
→ **Fix:** Add a `<a href="#main" class="skip-link">Skip to main content</a>` that's visually hidden but visible on focus.

**13. FAQ — 4 elements stuck at opacity 0**
The FAQ section has 4 children with `opacity: 0`, likely from scroll animations. If the animation doesn't trigger (e.g., slow connections, JavaScript disabled), the content stays invisible.
→ **Fix:** Set CSS fallback (`opacity: 1` as default) and use JS/IntersectionObserver only to add the animation class.

**14. Team page title — duplicated**
The title reads "Our Team | Loudoun Decks | Loudoun Decks" — "Loudoun Decks" appears twice.
→ **Fix:** Set it to "Our Team | Loudoun Decks".

**17. Structured Data (JSON-LD) — good but incomplete**
You have a valid LocalBusiness schema, which is great. It could be extended with `FAQPage` schema on the FAQ page, and `Service` schema on service pages.

**18. No Privacy Policy / Terms page**
No privacy policy or terms of service link anywhere. If you're using Google Analytics, cookies, or collecting data through the form, this is legally required.
→ **Fix:** Create and link Privacy Policy and Terms of Service pages in the footer and add data according to the business.
